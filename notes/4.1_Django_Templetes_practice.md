# Django Templates - Step by Step Implementation with Detailed Explanations

## Step 1: Create Django Project

```bash
django-admin startproject myproject
cd myproject
```

* **Explanation:**

  * `django-admin startproject myproject`: Creates a new Django project named `myproject`.
  * Project contains `manage.py` (CLI tool), project settings, WSGI/ASGI files.
  * `cd myproject`: Navigate inside the project folder to work further.
* **Mental model:** Think of a project as the **container or house** that will hold multiple apps (rooms).

## Step 2: Create Django App

```bash
python manage.py startapp myapp
```

* **Explanation:**

  * Creates a new Django app named `myapp`.
  * An app is a **self-contained module** with its own models, views, templates.
* **Mental model:** An app is a **room** inside the project (house) that handles specific functionality.

## Step 3: Configure App in Project

* Open `myproject/settings.py` and add `'myapp',` to `INSTALLED_APPS`:

```python
INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'myapp',  # our custom app
]
```

* **Explanation:**

  * `INSTALLED_APPS` lists all apps Django knows about.
  * Registering the app ensures Django can find models, templates, static files.
* **Mental model:** Registering an app is like **adding a new room to the house map** so the system knows it exists.

## Step 4: Create Template Folder

* Folder structure inside app:

```
myapp/
    templates/
        myapp/
            base.html
            home.html
            header.html
            footer.html
```

* **Explanation:**

  * Django looks for templates in `templates/<app_name>/`.
  * `base.html` is a reusable layout.
  * Other templates like `home.html` extend `base.html`.
* **Mental model:** Templates are **blueprints for HTML pages** where dynamic content can be inserted.

## Step 5: Create Base Template (`base.html`)

```html
<!DOCTYPE html>
<html>
<head>
    <title>{% block title %}My Site{% endblock %}</title>
    {% load static %}
    <link rel="stylesheet" href="{% static 'myapp/style.css' %}">
</head>
<body>
    <header>{% include 'myapp/header.html' %}</header>
    <main>{% block content %}{% endblock %}</main>
    <footer>{% include 'myapp/footer.html' %}</footer>
</body>
</html>
```

* **Explanation:**

  * `{% block title %}{% endblock %}`: Placeholder for title to be overridden.
  * `{% load static %}`: Load static files (CSS, JS, images).
  * `{% include 'header.html' %}`: Inserts reusable HTML snippet.
  * `{% block content %}{% endblock %}`: Placeholder for page-specific content.
* **Mental model:** `base.html` is the **master template or skeleton** that other pages fill in.

## Step 6: Create Home Template (`home.html`)

```html
{% extends 'myapp/base.html' %}

{% block title %}Home Page{% endblock %}

{% block content %}
<h1>Welcome, {{ name }}</h1>
<p>Your age is {{ age }}</p>

{% if age >= 18 %}
<p>You are an adult.</p>
{% else %}
<p>You are a minor.</p>
{% endif %}

<h2>Hobbies:</h2>
<ul>
{% for hobby in hobbies %}
    <li>{{ hobby|upper }}</li>
{% endfor %}
</ul>
{% endblock %}
```

* **Explanation:**

  * `{% extends 'base.html' %}`: Inherits the base template.
  * `{{ name }}`: Displays context variable passed from view.
  * `{% if %}` and `{% for %}`: Template tags for conditional logic and loops.
  * `|upper`: Template filter to convert text to uppercase.
* **Mental model:** This page **fills in the skeleton** provided by `base.html` with specific data.

## Step 7: Create Header & Footer Templates

* `header.html`:

```html
<h2>Reusable Header</h2>
```

* `footer.html`:

```html
<p>Reusable Footer</p>
```

* **Explanation:**

  * Reusable components included in multiple templates.
* **Mental model:** Think of them as **common pieces of furniture** reused across rooms.

## Step 8: Add Static Files

* Folder: `myapp/static/myapp/style.css`

```css
body {
    font-family: Arial, sans-serif;
    background-color: #f4f4f4;
}
h1 {
    color: #333;
}
```

* **Explanation:**

  * Static files are assets like CSS, JS, images.
  * Linked in templates using `{% static 'path/to/file' %}`.
* **Mental model:** Static files are **decoration and style** for your house (project).

## Step 9: Create View in `views.py`

```python
from django.shortcuts import render

def home(request):
    context = {
        'name': 'Bisham',
        'age': 20,
        'hobbies': ['Coding', 'Reading', 'Gaming']
    }
    return render(request, 'myapp/home.html', context)
```

* **Explanation:**

  * `render(request, template_name, context)`: Combines template with data.
  * `context` is a dictionary with variables for templates.
* **Mental model:** Views are **controllers** that prepare data and choose which blueprint (template) to render.

## Step 10: Configure URL

* `myapp/urls.py`:

```python
from django.urls import path
from . import views

urlpatterns = [
    path('', views.home, name='home'),
]
```

* `myproject/urls.py`:

```python
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('myapp.urls')),
]
```

* **Explanation:**

  * `urlpatterns` maps URLs to views.
  * `include('myapp.urls')` lets the app manage its own URLs.
* **Mental model:** URLs are **the road map or directions** to different rooms (views) in your project.

## Step 11: Run Development Server

```bash
python manage.py runserver
```

* Visit `http://127.0.0.1:8000/` to see your template in action.
* **Mental model:** Server is like **turning on the electricity** so your house (project) becomes functional.

---

### Key Concepts & Mental Models

1. **Project = House**: Holds multiple apps (rooms).
2. **App = Room**: Self-contained module with specific functionality.
3. **Template = Blueprint**: HTML skeleton with dynamic placeholders.
4. **Base Template = Master Skeleton**: Other templates extend it.
5. **View = Controller**: Decides what data to show and which template to render.
6. **URL = Road Map**: Connects web addresses to views.
7. **Static Files = Decoration**: CSS, JS, images that style and enhance pages.
8. **Context = Furniture**: Variables that fill templates with content.
9. **Template Tags & Filters = Tools**: Logic and formatting inside templates.
